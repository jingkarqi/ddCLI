# ddCLI 指令交互逻辑设计

## 核心命令

### dd <自然语言命令>
解析用户输入的自然语言并执行相应的命令

示例:
- `dd 删除当前目录下的所有txt文件`
- `dd 创建一个名为test的新文件夹`

交互逻辑:
1. 用户输入自然语言命令
2. 系统调用AI API将自然语言转换为实际命令
3. 显示生成的命令预览
4. 对于危险命令，高亮显示并要求用户确认
5. 用户确认后执行命令
6. 返回命令执行结果

## 配置相关命令

### dd config [选项]
配置管理命令

子命令:
- `dd config show`: 显示当前所有配置
- `dd config reset`: 重置为默认配置

交互逻辑:
- `dd config show`: 直接显示当前配置信息
- `dd config reset`: 提示用户确认是否重置配置，确认后执行重置操作

### dd security [选项]
安全设置管理命令

子命令:
- `dd security whitelist`: 管理命令白名单

交互逻辑:
- `dd security whitelist`: 进入白名单管理界面，允许用户添加或删除白名单命令

## 系统命令

### dd help [命令]
帮助系统

子命令:
- `dd help`: 显示总体帮助信息
- `dd help <命令>`: 显示特定命令的详细帮助
- `dd help examples`: 显示常用示例

交互逻辑:
- 根据用户输入的具体help选项显示相应的帮助信息

### dd version
版本信息

交互逻辑:
1. 显示当前版本号
2. 显示更新日志摘要
3. 检查并显示更新状态

### dd setting
交互式设置

交互逻辑:
1. 执行后显示设置菜单:
   ```
   ddCLI 设置选项:
   -------------------
   1. API配置
   2. 安全设置
   3. 命令历史
   4. 重置配置
   5. 退出设置

   请使用上下箭头选择设置项，按Enter确认:
   ```
2. 用户使用上下箭头选择选项
3. 按Enter进入相应设置界面
4. 在各设置界面中进行具体配置
5. 选择"退出设置"返回主命令行

### dd history
命令历史

交互逻辑:
1. 显示历史命令记录列表
2. 显示各命令的使用频率统计
3. 提供选项清除历史记录
